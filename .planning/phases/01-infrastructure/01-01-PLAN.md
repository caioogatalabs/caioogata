  <action>
1. Update dependencies in package.json:
   - Remove from devDependencies: `tailwindcss` (v3), `autoprefixer`, `postcss`
   - Add to devDependencies: `tailwindcss` (^4.0.0), `@tailwindcss/postcss` (^4.0.0)
   - Keep `postcss` as it is needed by Next.js, but update to latest if needed
   - Run `pnpm install` after changes

2. Update postcss.config.mjs:
   Replace the entire contents with:
   ```js
   /** @type {import('postcss-load-config').Config} */
   const config = {
     plugins: {
       '@tailwindcss/postcss': {},
     },
   }
   export default config
   ```
   Note: autoprefixer is no longer needed — Tailwind v4 includes it.

3. Update src/app/globals.css:
   - Remove the three `@tailwind base;`, `@tailwind components;`, `@tailwind utilities;` directives (lines 1-3)
   - Replace with a single `@import "tailwindcss";` at the top of the file
   - Keep all existing `:root` custom properties, base styles, keyframes, and utility classes intact below the import

4. Handle tailwind.config.ts:
   - In Tailwind v4, configuration moves to CSS. The existing `tailwind.config.ts` content (colors, fonts, spacing, etc.) will be superseded by the token system in Plan 02.
   - For now, delete `tailwind.config.ts` — Tailwind v4 does not use JS config files by default. Content detection is automatic in v4 (scans all template files).
   - The V1 theme values (colors, fonts, spacing) in the old config are not needed yet — they will be replaced by the design token system in the next plan.

5. Run `pnpm build` to verify the project builds successfully with Tailwind v4.
   - If build fails due to missing utility classes or changed syntax, fix the specific issues:
     - `@layer utilities` blocks may need adjustment
     - `@apply` should still work in v4
     - Any v3-specific plugin syntax needs updating
  </action>
