   - At the top of the file, structure imports in this order:
     ```css
     @import "tailwindcss";
     @import "../tokens/index.css";
     ```
   - After the imports, add the @font-face declaration:
     ```css
     @font-face {
       font-family: 'Fabio XM';
       src: url('../fonts/FabioXM-Variable.ttf') format('truetype');
       font-weight: 300 900;
       font-style: normal;
       font-display: swap;
     }
     ```
   - Remove the old V1 `:root` block (lines 5-28 in current file) that defines `--color-primary`, `--color-secondary`, `--bg-neutral`, `--text-primary`, etc. These are replaced by the token system's primitive and semantic tokens.
   - Remove the old `--section-gap` media query block as well (spacing is now in tokens).
   - Update the `body` selector to use token-based font:
     ```css
     body {
       background-color: var(--color-bg);
       color: var(--color-text-primary);
       font-family: var(--font-sans);
       -webkit-font-smoothing: antialiased;
       -moz-osx-font-smoothing: grayscale;
       -webkit-text-size-adjust: 100%;
       touch-action: manipulation;
     }
     ```
   - Update `::selection` to use token colors:
     ```css
     ::selection {
       background-color: var(--color-bg-fill-primary);
       color: var(--color-text-on-primary);
     }
     ```
   - Keep all other styles (focus, sr-only, reduced-motion, keyframes, utilities) but update any hardcoded V1 colors to use token variables where they appear:
     - `.editor-slider` thumb background `#FAEA4D` becomes `var(--color-bg-fill-primary)`
     - `.editor-slider` border `#454546` becomes `var(--color-border-secondary)`
     - `.editor-slider::-moz-range-track` background `#454546` becomes `var(--color-border-secondary)`
     - `.editor-slider::-moz-range-progress` background becomes `var(--color-bg-fill-primary)` with 0.4 opacity via oklch alpha

3. Update `src/app/layout.tsx`:
   - If it currently imports `@fontsource/cascadia-mono` or uses `next/font` for Cascadia Mono, keep that for the mono font
   - Ensure the body or html element does NOT have a hardcoded font-family class that would override the CSS — the font-family is set in globals.css via `var(--font-sans)`
   - If layout.tsx sets a CSS variable like `--font-jetbrains` via next/font, remove that pattern since fonts are now managed via @font-face in globals.css and token variables

4. Run `pnpm build` to verify everything compiles.
