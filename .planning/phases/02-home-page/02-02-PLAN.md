**Create `src/components/sections/v2/StickyHeader.tsx`:**

A `'use client'` component that appears as a sticky bar when the user scrolls past the Intro section.

**Structure:**
```
<header
  role="banner"
  className="sticky top-0 z-50 h-14 transition-opacity duration-300"
  style={visible ? { opacity: 1 } : { opacity: 0, pointerEvents: 'none' }}
>
  <div> (flex row, justify-between, items-center, full height, grid-margin padding)
    LEFT: "CO" logo (same as intro, Fabio XM Bold 20px)
    RIGHT: flex row gap-3 items-center
      - "V2.0.01" (overline style, mono -- NOT Fabio XM)
      - "Built for human and AI assistance" (overline style -- NOT Fabio XM, hidden on mobile)
      - "Ask about" CTA pill (same as intro CTA)
  </div>
</header>
```

**Visual specs per UI-SPEC:**
- Height: 56px (h-14)
- Background: `--color-bg-surface-secondary` at 60% opacity with `backdrop-filter: blur(20px)` (glassmorphism per Design.md)
- CSS: `background: oklch(0.15 0 0 / 0.6)` or use the token with opacity
- Z-index: 50
- Position: `sticky; top: 0`

**Visibility logic:**
- Track scroll position with `useState` + `useEffect` with scroll listener
- Show header when `window.scrollY > window.innerHeight * 0.6` (past 60% of intro)
- Use `opacity` transition (0.3s ease-smooth) for smooth crossfade, NOT display:none
- When hidden: `pointer-events: none` so it doesn't block interaction

**prefers-reduced-motion for opacity transition (CRITICAL):**
- The opacity is controlled via inline `style` prop, so the globals.css `@media (prefers-reduced-motion)` rule does NOT cover it
- Check `window.matchMedia('(prefers-reduced-motion: reduce)')` on mount (store in useRef)
- When reduced motion is preferred: apply opacity changes instantly (no CSS transition class -- remove `transition-opacity duration-300` from className, or set `transition: none` in the inline style)
- Implementation: conditionally include `transition-opacity duration-300` in className based on the media query result

**Props:** None needed -- reads scroll position internally.

**Integrate with IntroSection.tsx:**
- Import StickyHeader inside IntroSection.tsx (or inside PageShell -- whichever is cleaner)
- The recommended approach: render StickyHeader as a sibling ABOVE IntroSection in PageShell.tsx, OR render it inside IntroSection.tsx above the section element
- If rendered inside IntroSection, place it before the `<section>` so it stacks naturally
- Since PageShell already imports IntroSection, the simplest integration: update IntroSection.tsx to also render StickyHeader at the top

**Responsive:**
- Desktop: logo + version + tagline + CTA
- Tablet: logo + CTA only (hide version and tagline)
- Mobile: logo + CTA only

**Accessibility:**
- `role="banner"` on the header
- When hidden (opacity 0), set `aria-hidden="true"`
