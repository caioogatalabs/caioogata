Create `src/hooks/useMenuNavigation.ts` -- a `'use client'` hook that manages menu state: active index, filter text, filtered items list, and keyboard event handling.

**Interface:**
```typescript
interface UseMenuNavigationOptions {
  items: MenuItem[]
  onSelect: (item: MenuItem) => void
  onEscape?: () => void
}

interface UseMenuNavigationReturn {
  activeIndex: number
  setActiveIndex: (index: number) => void
  filterText: string
  setFilterText: (text: string) => void
  filteredItems: MenuItem[]
  inputRef: React.RefObject<HTMLInputElement>
  hoveredIndex: number | null     // mouse hover tracking (separate from keyboard active)
  setHoveredIndex: (index: number | null) => void
}
```

**Behavior:**
1. **Filter**: `filterText` filters `items` by matching against `label` or `description` (case-insensitive includes). Empty filter = all items.
2. **Arrow keys**: `ArrowDown` increments activeIndex (wraps to 0), `ArrowUp` decrements (wraps to end). Only when input is focused or body has focus.
3. **Enter**: calls `onSelect(filteredItems[activeIndex])`.
4. **Escape**: calls `onEscape()` if provided. Per D-11: navigation always returns to home.
5. **Type-to-filter**: any character typed while section is visible focuses the input and updates filter. Use a `useEffect` with `keydown` listener on `document`.
6. **Active index resets to 0** when filter changes (new filtered list).
7. **Hovered index** is separate from activeIndex -- mouse hover sets hoveredIndex, keyboard nav sets activeIndex. The "highlighted" item is `hoveredIndex ?? activeIndex`.

**Implementation notes:**
- Use `useState` for activeIndex, filterText, hoveredIndex
- Use `useMemo` for filteredItems
- Use `useEffect` for document-level keydown listener
- Use `useRef` for input element
- Use `useCallback` for onSelect handler
- Do NOT use `useRouter` in this hook -- keep it pure state. The consumer (MenuSection) handles routing.

Create `src/components/sections/v2/FloatingPreview.tsx` -- a `'use client'` component that renders a fixed-position image that follows the cursor with velocity-based skew.

**Props:**
```typescript
interface FloatingPreviewProps {
  imageSrc: string | null    // null = hidden
  alt: string
  mouseX: number
  mouseY: number
  velocityX: number          // horizontal velocity for skew
}
```

**Rendering:**
- `position: fixed`, `pointer-events: none`, `z-index: 100`
- Width: 500px, aspect ratio ~5:3 (height ~300px)
- Border radius: `--radius-component-md` (12px)
- Overflow hidden (clip the image)
- Image: `<img>` with `object-fit: cover`, full width/height
- Transform: `translate(${mouseX}px, ${mouseY}px)` with offset so image appears beside cursor (not centered on it -- offset ~20px right, ~20px up)
- Skew: `skewX(${clamp(velocityX * 0.1, -30, 30)}deg)` based on horizontal velocity (D-18)
- Opacity: 1 when imageSrc is truthy, 0 when null -- transition 0.2s ease-smooth
- Scale: enter at `scale(0.8)`, animate to `scale(1)` on show -- transition 0.3s ease-out

**prefers-reduced-motion (CRITICAL):**
- Check `window.matchMedia('(prefers-reduced-motion: reduce)')` on mount (use a `useRef` or `useState` for the media query result)
- When reduced motion is preferred: set `skewX` to `0deg` (no velocity skew), set `scale` transitions to `none` (instant show/hide), keep opacity transition but reduce to 0.1s
- This is necessary because skewX and scale are applied via inline styles, so the CSS `@media (prefers-reduced-motion)` rule in globals.css does NOT cover them

**Performance:**
- Use CSS custom properties `--preview-x` and `--preview-y` for position, updated via `style` prop
- Use `will-change: transform` on the container
- The transform uses CSS `translate3d` for GPU acceleration

**Image placeholder:**
- For now, use a solid `--color-bg-surface-primary` background as fallback when no preview image exists for a menu item
- Preview images will be added to `public/previews/{key}.webp` -- the component just needs to handle the `imageSrc` prop

**Hidden on mobile:**
- Add `hidden lg:block` to only show on desktop (>=1024px per UI-SPEC: no previews on tablet)
