## 5. Gaps identificados — portfólio atual vs. mercado

| Elemento | Portfólio atual | O que o mercado usa | Gap |
|---|---|---|---|
| Tagline/headline | "Design Director specializing in…" | "Design Engineer" / "UX Engineer" / "Design Systems Lead" | Não aparece nos resultados de busca de Design Engineer |
| Abertura do bio | "I'm a Design Director" | Abre com o que faz, não o título | Título-first reduz match em buscas de IC |
| JavaScript | "JavaScript (intermediate)" | Skill omitida ou declarada como proficiency | "Intermediate" sinaliza insuficiência para Design Engineer |
| Vue.js / TypeScript | Não listados | Obrigatórios em vagas de Design Engineer e DS | Ausentes — Console Kit é Vue 3 + TypeScript |
| "Front-end Collaboration" | Skill listada assim | Mercado usa "UI Engineering" ou "Frontend Development" | Terminologia diferente, não match em buscas |
| AI-native workflows | Claude AI listado como tool | Ramp/2026: "Cursor · Claude Code · AI-native" como skill | Subcategorizado — deveria ser posicionamento central |
| Console Kit | 3º projeto listado | Deveria ser âncora — Vue 3, TypeScript, 6k commits, OSS | Enterrado, não lidera o posicionamento |
| DevEx | No título (Azion) mas não no tagline | Recrutadores buscam "DevEx" / "Developer Experience Designer" | Não aparece onde é pesquisado |
| Open source | Citado nas conquistas | OKX/2026: "open source contributions" como diferencial explícito | Poderia ter mais destaque como keyword |
| Design Tokens | Listado como skill (90%) | Obrigatório em DS e Design Engineer — ok | Bem posicionado, manter |
| Atomic Design | Listado em Methodologies | Obrigatório em DS — ok | Bem posicionado |
| PLG Strategy | Listado | Não aparece em nenhuma JD como keyword principal | Pode ser removido ou rebaixado |

## 7. Linguagem a adotar — exemplos de frases das JDs

Estas frases aparecem em múltiplas JDs de alta qualidade e devem ressoar no conteúdo do portfólio e LinkedIn:

- **"Design and build"** (em vez de "design and collaborate with developers")
- **"End-to-end ownership"** (em vez de "led design process")
- **"Ship to production"** (em vez de "delivered designs")
- **"High-craft product experiences"** (Ramp — linguagem de nível de qualidade)
- **"Prototype in code"** (Stripe/Ramp — habilidade diferencial)
- **"Bridge design and engineering"** (Cloudflare — posicionamento híbrido)
- **"Scale design systems"** (Google/StubHub — não só "build")
- **"Hands-on when needed"** (Nubank — evita percepção de gestor distante do craft)
- **"Strategic and tactical"** (Nubank — ambidestria de senioridade)
- **"Drive consistency, efficiency, and quality"** (OKX — tríade de DS)
- **"Open source"** + número de contribuidores/commits (diferencial verificável)

## 8. O que NÃO usar (terminologia que não aparece nas JDs)

- ~~"Art Direction" como skill principal~~ (soa como advertising, não tech)
- ~~"PLG Strategy" como expertise central~~ (não aparece como keyword de busca)
- ~~"JavaScript (intermediate)"~~ (sinaliza inadequação; omitir ou reformular)
- ~~"Front-end Collaboration"~~ (usar "UI Engineering" ou "Frontend Development")
- ~~Percentuais de skill (95%, 90%)~~ (não é prática de mercado tech; usar nível ou omitir)
- ~~"20 years in Art Direction"~~ (como abertura; reposicionar como contexto, não headline)

---

## 9. Prioridade de ajustes (ordem de impacto)

1. **Headline/tagline** — incluir "Design Engineer" e/ou "DevEx" para aparecer em buscas
2. **Abertura do bio** — começar com o que faz (Console Kit, DevEx, Design Systems) não com o título
3. **Skills técnicas** — adicionar Vue.js, TypeScript; reformular JavaScript; renomear "Front-end Collaboration"
4. **Posição do Console Kit** — mover para primeiro projeto; é o maior diferencial técnico
5. **AI-native** — elevar Cursor + Claude Code de "tools" para posicionamento estratégico
6. **Open source** — destacar como keyword e conquista verificável
7. **DevEx** — garantir que aparece no tagline ou headline do LinkedIn
